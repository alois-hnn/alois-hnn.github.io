Die unteren drei Schichten des TCP/IP-Modells, die Netzzugangsschicht, die Internetschicht und die Transportschicht, bilden die Grundstruktur für die Protokolle, die zum Senden von Daten über ein Netzwerk verwendet werden. In der Anwendungsschicht werden die in den unteren drei Schichten festgelegten Regeln weiter verfeinert, um eine Verbindung zwischen bestimmten Computerprogrammen zu ermöglichen. Ein Beispiel hierfür ist FTP, eine Programmierschnittstelle für den Austausch von Dateien über ein Netzwerk.

Wie diese Beispiele verdeutlichen, ermöglicht TCP/IP die Kommunikation zwischen verschiedenen Computeranwendungen über seine definierten Protokolle auf der Anwendungsschicht. Die Protokolle der Anwendungsschicht haben einen großen Einfluss auf den Betrieb eines Netzes, da sie die Art und Weise der Interaktion zwischen den Anwendungen bestimmen. Es ist wichtig, sich über die verwendeten Protokolle auf dem Laufenden zu halten und solche zu vermeiden, die veraltet sind oder als unsicher gelten. So sind beispielsweise das File Transfer Protocol (FTP) und Telnet Beispiele für Protokolle, die aufgrund potenzieller Sicherheitsrisiken und Anfälligkeiten für Angriffe nicht verwendet werden sollten (De Paz, 2017). Außerdem sollten stattdessen modernere Protokolle wie Secure File Transfer Protocol (SFTP) und Secure Shell (SSH) verwendet werden, um die Sicherheit zu gewährleisten. Es ist also wichtig, nur die aktuellsten Protokolle zu verwenden, damit die Netzwerke optimal und sicher funktionieren.

De Paz, A. (2017, April 03). Die 7 wichtigsten Arten von Netzwerkprotokollen. Abgerufen von https://www.liquidweb.com/blog/the-7-most-important-types-of-network-protocols/

Die Anwendungsschicht ist verantwortlich für die Bereitstellung einer Plattform für High-Level-Anwendungen wie E-Mail, Dateiübertragung und World Wide Web, um eine durchgängige Kommunikation zu gewährleisten. Ein Beispiel für eine Verbindung der Anwendungsschicht ist eine Web-Browsing-Sitzung, bei der ein Benutzer Daten von Websites anfordert und empfängt.

Das Transmission Control Protocol / Internet Protocol (TCP/IP) ist ein grundlegendes Modell für die Datenübertragung, das heute in praktisch der gesamten digitalen Kommunikation verwendet wird. Es besteht aus einem hierarchischen System von Schichten, das die Datenkommunikation in überschaubaren und verständlichen Anweisungen organisiert.

Die Anwendungsschicht bildet die oberste Ebene des TCP/IP-Modells und ist für die Aktivitäten im Zusammenhang mit bestimmten Protokollen (z. B. HTTP, FTP, SMTP) zuständig. Ein Beispiel dafür, wie die Anwendungsschicht funktioniert, sind zwei Computer. Mit Hilfe eines Webbrowsers muss der Client eine Webseite vom Server "herunterladen". Dazu erstellt der Client ein HTTP-Anforderungspaket, das die URL der Webseite enthält. Dieses Anforderungspaket wird an den Server gesendet, der ein HTTP-Antwortpaket mit der Webseite an den Client zurücksendet (Halsall, 2020). Über eine Programmierschnittstelle (API) kann auf das zugrunde liegende TCP/IP-Modell zugegriffen werden, so dass Entwickler Web- oder Netzwerkanwendungen erstellen können. Die WinSock-API von Microsoft ist ein weit verbreitetes Beispiel für eine Programmierschnittstelle zum TCP/IP-Modell (Schneider, 2019).

Schneider, G. (2019, October 2). Eine kurze Einführung in TCP/IP und die Anwendungsschicht. Abgerufen von https://www.geeksforgeeks.org/a-brief-introduction-to-tcpip-and-application-layer/

Auf der Anwendungsschicht werden Protokolle wie Hypertext Transfer Protocol (HTTP), Simple Mail Transfer Protocol (SMTP), File Transfer Protocol (FTP) und Domain Name System (DNS) zur Interaktion mit Benutzern und anderen Anwendungen verwendet.

So stellt beispielsweise eine Anwendung auf einem iPad über HTTP eine Verbindung zu einer Website her, und die Website verwendet SMTP, um eine E-Mail an ihre Benutzer zu senden. Um diese Verbindung herzustellen, muss die Anwendungsschicht die spezifische IP-Adresse der Website oder des E-Mail-Servers kennen, und das TCP/IP-Modell liefert die Informationen, die für die Kodierung und Dekodierung der Daten erforderlich sind. Darüber hinaus kann eine Programmierschnittstelle verwendet werden, um mit dem TCP/IP-Modell zu interagieren und Daten zu senden und zu empfangen. Die Berkeley Sockets API ist eine von vielen solchen Programmierschnittstellen, die Programmen den Zugriff auf das TCP/IP-Modell ermöglicht. Das TCP/IP-Modell ist somit grundlegend für unser Verständnis der Kommunikation im Internet.
