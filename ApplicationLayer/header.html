Header

Jedes Datenpaket besteht aus einem Header und einem Payload. Der Header enthält Informationen über den Server (Sender- und Empfängeradressen), sowie über den Payload. Auch welche Protokolle zur Verbindung verwendet werden, wird durch den Header ersteinmal definiert. Die Umsetzung erfolgt später durch die anderen Schichten der Modelle. Die Anwendungsschicht charakterisiert somit die Daten. Sie stellt den Payload für den Nutzer zur Verfügung und bestimmt wie dieser zwischen Client und Server ausgetauscht werden soll. Die Umsetzung erfolgt schließlich basierend auf diesen Informationen, auf den anderen Schichten.

Der Screenshot stammt aus den Dev-Tools einer Website und zeigt im Grunde, was die Anwendungsschicht macht und welche Information auf welche Art und Weise verarbeitet werden.
Zu den generellen Informationen zählen die Request URL, sprich das was man in der Adresszeile des Web-Browsers eingibt, Statuscodes, sowie die IP-Adresse des Webservers. Zu diesem findet man im Response Header weitere Informationen. So zeigt der Punkt Server welcher Webserver verwendet wird, um welchen Content-Type es sich handelt, in diesem Fall der Seitenquelltext und viele weitere relevante Parameter, welche für den Client wichtig sind, um mit dem Server über das Netzwerk kommunizieren zu können.
Über den Client liefert der Request Header Informationen. Welcher Content-Type wird als Response akzeptiert? Welche Sprachen soll dieser haben? Welches Betriebssystem und Browser wird verwendet?

Dieser Header ist ein weiteres Beispiel, allerdings für eine Websocket Verbindung. Auch für diese muss festgelegt werden, wie die Kommunikation ablaufen soll. Zudem zeigt der Browser nicht nur Informationen zur Request, sondern auch zur Response. Dadurch ist es möglich die Art und den Erfolg des Datenaustauschs von Server und Client bidirektional nachzuvollziehen.
